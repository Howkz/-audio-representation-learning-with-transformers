# Launch One Experiment at a Time (E00 -> E11)

## Quick start

From the project root (`daaa-project-reborn`):

```bash
pip install -r requirements.txt
bash scripts/experiments/run_E00.sh
```

## Available scripts

- `run_E00.sh`
- `run_E01.sh`
- `run_E02.sh`
- `run_E03.sh`
- `run_E04.sh`
- `run_E05.sh`
- `run_E06.sh`
- `run_E07.sh`
- `run_E08.sh`
- `run_E09.sh`
- `run_E10.sh`
- `run_E11.sh`

Each script supports:

- `--resume` to continue an interrupted run
- `--dry-run` to validate the execution plan without training
- `--bootstrap-prereqs` (only useful for `E09`/`E10`/`E11`)

Examples:

```bash
bash scripts/experiments/run_E04.sh --resume
bash scripts/experiments/run_E09.sh --dry-run
bash scripts/experiments/run_E09.sh --bootstrap-prereqs --resume
```

## Important notes

- `E09`, `E10`, `E11` depend on prior screening results (`E01`..`E08`).
- Use `--bootstrap-prereqs` to auto-run missing screening prerequisites before a final run.
- Selection runs (`SEL01`..`SEL05`) are auto-generated by the suite runner.
- Main outputs are written under:
  - `results/experiments/<EXP_ID>/...`
  - `results/suite/leaderboard_screening.csv`
  - `results/suite/leaderboard_selection.csv`
  - `results/suite/suite_summary.csv`
